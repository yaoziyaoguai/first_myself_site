# CLAUDE.md

## 项目概述
这是一个面向数据仓库工程师的个人网站项目。
项目目标不是炫技，而是通过真实项目学习 vibe coding，并产出一个可上线、可维护、可持续迭代的个人站点。

## 产品目标
- 清晰展示我的职业身份和技术方向
- 展示项目经历、技术文章和思考
- 优先保证简单、稳定、可维护
- 优先完成并上线一个可用版本，而不是一开始做复杂系统

## 技术栈
- Next.js
- TypeScript
- Tailwind CSS
- shadcn/ui
- MDX（用于博客内容）
- Vercel（用于部署）

## 编码原则
- 优先选择简单、可读性高的实现，不要炫技
- 保持组件职责清晰，尽量小而可复用
- 避免过度设计
- 除非我明确提出，否则不要引入后端服务
- 优先使用本地内容或静态内容
- 遵循响应式设计和基础可访问性原则
- 尽量使用语义化 HTML

## UI 风格
- 风格简洁、现代、专业
- 页面布局清爽，有信息层次
- 可以有一点“数据 / AI / 工程感”，但不要花哨
- 优先保证阅读体验，而不是复杂特效
- 整体气质要符合数据工程师 / AI 工程方向的个人品牌

## 内容策略
- 首页要快速说明“我是谁、做什么、擅长什么”
- 项目页要强调：背景、职责、方法、结果
- 博客内容要适合技术写作阅读
- 不要编造虚假经历、虚假公司、虚假指标或虚假成果

## 文件与架构规则
- 路由页面放在 app/
- 可复用组件放在 components/
- 内容文件放在 content/
- 工具函数放在 lib/
- 不要把无关逻辑塞进 page 组件
- 展示逻辑和内容解析逻辑尽量分离

## 工作方式
- 在实现一个功能之前，先简要说明你的实现思路
- 完成后，说明你改了哪些文件、实现了什么
- 对于非 trivial 的改动，请同步更新 README 或相关注释
- 采用小步迭代方式：一次完成一个功能，再进入下一个
- 不要为了“顺手”去重构无关文件，除非确实必要

## 质量要求
- 页面必须可以正常渲染，不能有明显运行时报错
- 页面需要同时适配桌面端和移动端
- 避免死代码和明显无用代码
- TypeScript 类型尽量清晰，不要大量使用 any
- 在完成重要改动后，运行 lint 或相关检查

## MVP 阶段禁止事项
- 不要引入数据库
- 不要引入登录 / 鉴权系统
- 不要引入 CMS
- 不要做复杂动画
- 不要在最终页面中保留明显无意义的占位内容，除非明确标注为占位

## 当前 MVP 范围
本项目第一阶段只实现以下页面：
- Home
- About
- Projects
- Blog
- Contact

第一阶段优先完成：
1. 项目初始化
2. 全局布局
3. 导航栏和页脚
4. 各页面基础结构
5. 项目列表展示
6. 博客列表和文章详情
7. 基础响应式适配
8. 部署上线

## 沟通要求
- 解释尽量用中文
- 代码、命令、路径、技术名可以保留英文
- 如果你发现我的要求存在冲突、范围过大或实现成本明显偏高，请直接指出并给出更稳妥的替代方案
